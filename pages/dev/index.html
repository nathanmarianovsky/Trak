Search Bar Features

The search bar on the home page of the application has been designed to provide mechanisms for performing searches based upon any record parameter desired. More specifically, the search bar may be utilized in the following two manners:
  • By typing any string without the @ symbol will initiate a simple search based on finding records whose name includes the desired string.
  • Provided the search @option:param the application will filter all records whose attribute "option" matches "param." More specifically:
    • The "option" of a search can correspond to any of the following: category, name, genres, jname, ogname, demographic, writers, authors, illustrators, directors, producers, musicians, studio, publisher, jpublisher, license, isbn, or media.
    • For the above options jname, ogname, musicians, jpublisher, and media reference the Japanese name, original title, musical producers, Japanese publisher, and media type of a record, respectively.
    • Multiple searches can be accomplished at the same time by formatting the search in the format @option1:param1,@option2:param2,...,@optionK:paramK for K many searches.
    • For any single search multiple parameters can be provided by stating whether they should be searched in conjunction or disjunction with each other. More specifically:
      • The & symbol is used to denote the operation of conjunction, i.e. @option:param1&param2 will filter the records whose attribute "option" includes both "param1" AND "param2."
      • The | symbol is used to denote the operation of disjunction, i.e. @option:param1|param2 will filter the records whose attribute "option" includes either "param1" OR "param2."
      • The & operator takes precedence over the | operator, i.e. @option:param1&param2|param3 will filter the records whose attribute "option" includes param1 AND (param2 OR param3).

    In order to export the checked library records simply provide the directory to which the xlsx or zip files should be saved.

    For the case of a xlsx file either a simple file can be generated containing a single worksheet with all basic record details or a detailed file containing the same information provided by the simple version along with a separate worksheet for each record containing details such as season information, episode information, etc... If desired, any zip file generated in the export process can be compressed by checking the "Compress" option below.

    Directory:

    To import into the library records simply select the zip files which contain the data and click to import. Any zip file to be imported should correspond to an export generated from this application. A sample can be found here.

    To import into the library records simply select the xlsx files which contain the data and click to import. The instructions for how the xlsx files should be formatted is described for each record option above.

    To import anime records into the library via a simple xlsx file there should be a worksheet titled "Category-Anime" whose columns are expected to be in the order:

    | Name | Japanese Name | Rating | Comments/Review | Synopsis | Directors |
    | Producers | Writers | Music Directors | Studios | Licensed By | Release Date | Genres |
    The first row should be used for column headers with data starting on the second row whose entries can consist of a comma separated list for multiple values in a single entry. Select the xlsx files which contain the data and click to import. A sample can be found here.

    To import anime records into the library via a detailed xlsx file multiple worksheets are expected to be provided where the first worksheet corresponds to the same data as with a simple xlsx import. All other worksheets are expected to be titled in the format "Anime-FormattedName" where FormattedName is the value in the first column of the "Category-Anime" worksheet whose words all start with a capital letter and has all whitespaces removed. The columns for these same worksheets are expected to be in the order:

    | Type | Name | Start Date | End Date | Status | Last Watched |
    | Rating | Episode # | Episode Name | Comments/Review |
    The first row should be used for column headers with data starting on the second row. Furthermore, the value of a record's related content type can be either a "Season" or "Single" where single corresponds to an entry with a single episode and status can take on one of five values "Watching", "Completed", "On-Hold", "Dropped", or "Plan To Watch". For type single provide the release date under the start date column and ignore the columns corresponding to the episode number and episode name. Select the xlsx files which contain the data and click to import. A sample can be found here.

    To import book records into the library via a simple or detailed xlsx file there should be a worksheet titled "Category-Book" whose columns are expected to be in the order:

    | Title | Original Title | Rating | Comments/Review | Synopsis | ASIN/ISBN | Authors |

    The first row should be used for column headers with data starting on the second row whose entries can consist of a comma separated list for multiple values in a single entry. Select the xlsx files which contain the data and click to import. A sample can be found here.
